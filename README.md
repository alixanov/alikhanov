# ALIKHANOV — SaaS / CRM / ERP / Telegram-боты

Сайт услуг фриланс-разработчика (Узбекистан) на Next.js (App Router), с поддержкой RU/UZ/EN.

## Стек

- Next.js (App Router, Turbopack)
- next-intl (i18n: `ru` / `uz` / `en`, всегда с префиксом локали, `uz` — локаль по умолчанию)
- Обычный CSS по компонентам (без Tailwind/UI-кита)
- AOS (скролл-анимации), Swiper (карусель кейсов)
- Контакт-форма → собственный Telegram-бот (без сторонних форм-сервисов)

## Архитектура

Весь сайт — **одна страница** (`app/[locale]/page.js`), все секции стоят друг под другом со скроллом по клику на навигацию. Реальных подстраниц `/services`, `/work` и т.д. **нет** — это осознанное решение (пробовали разбивать на настоящие роуты дважды, оба раза вернули обратно на одну страницу).

При клике по навигации URL в адресной строке косметически меняется на `/uz/services` и т.п. через `history.pushState` (без реальной навигации) — см. `components/layout/Header.jsx`. Middleware (`proxy.js`) делает rewrite: прямой заход или обновление страницы на такой "косметический" адрес отдаёт ту же единую страницу вместо 404.

## Секции страницы (по порядку)

1. **Hero** — заголовок, кнопки, мини-форма заявки
2. **Услуги** (`Services`) — 4 карточки (SaaS, CRM, ERP, Telegram-боты) с модалками
3. **Кейсы** (`Work`) — карусель, 3 реальных проекта: Parvoz Online, MyEduCheck, DeluxPC
4. **Отзывы** (`Testimonials`) — бегущая строка отзывов
5. **Обо мне** (`About`) — фото, статистика, навыки, опыт/образование
6. **CTA-баннер** — призыв к действию перед формой
7. **Контакты** (`Contact`) — прямые контакты + форма

## Структура репозитория

- `app/[locale]/page.js` — вся страница целиком
- `app/api/contact/route.js` — приём заявок из формы, отправка в Telegram-бота (honeypot + rate-limit + валидация длины полей)
- `proxy.js` — middleware: i18n-роутинг next-intl + rewrite косметических URL секций
- `content/` — тексты услуг, кейсов, отзывов, общих данных (по языкам)
- `components/` — переиспользуемые компоненты (layout, секции, форма, аналитика)
- `messages/{ru,uz,en}.json` — короткие UI-строки (навигация, кнопки, лейблы)

## Переменные окружения

Скопируйте `.env.example` в `.env.local` и заполните (значения — только у себя локально/в Vercel, **не коммитить**):

```
TELEGRAM_BOT_TOKEN=       # токен бота от @BotFather
TELEGRAM_CHAT_ID=         # ваш chat_id (бот должен получить от вас хотя бы одно сообщение до первой отправки)
NEXT_PUBLIC_YANDEX_METRICA_ID=   # опционально
NEXT_PUBLIC_GA_ID=               # опционально
```

В Vercel эти переменные добавляются отдельно через Project Settings → Environment Variables — `.env.local` в проде не используется.

## Безопасность формы обратной связи

- **Honeypot** — скрытый чекбокс (`hpAgree`), не текстовое поле: у текстового поля с именем вроде "company"/"website" браузер может сам подставить сохранённые данные автозаполнения, из-за чего настоящая заявка ошибочно засчитывается как бот.
- **Rate-limit** — не более 5 заявок с одного IP за 10 минут (best-effort, в памяти процесса).
- **Валидация длины** полей (имя/email/сообщение), значения обрезаются (`trim`) перед проверкой.

## Запуск


## Обновление контента

- Тексты услуг — `content/services/*.js`
- Кейсы — `content/case-studies/*.js` (картинки — `public/images/case-studies/`)
- Отзывы — `content/testimonials/data.js` (**3 из 4 сейчас временные плейсхолдеры**, отмечены комментарием `TEMPORARY` в коде — заменить на реальные, когда появятся)
